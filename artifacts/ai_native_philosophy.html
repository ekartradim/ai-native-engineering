Philosophy

6 principles
always enforced
no exceptions
01
plan
Plan first,
always
always enforced
Write a detailed plan before writing any code. The plan defines exact file paths, logic, and verification steps. No implementation begins without a written, approved plan.
02
verify
Test-driven
development
always enforced
Write and run tests, always. Red before green — a failing test must be observed before code is written to pass it. Code written before tests is deleted, no exceptions.
03
process
Systematic
over ad-hoc
process over guessing
Follow the defined process even when it feels slower. Guessing and improvising create invisible debt. A repeatable system compounds in reliability over time.
04
design
Complexity
reduction
simplicity is primary goal
Simplicity is the primary design goal, not a nice-to-have. When two solutions work, choose the simpler one. Complexity is a liability that must be consciously accepted, never defaulted into.
05
integrity
Evidence
over claims
verify before declaring done
Never declare success without evidence. Tests must pass, outputs must be observed, verification steps must be executed. Claims without evidence are not progress.
06
learn
Compound
knowledge
always save lessons learned
Save lessons learned to structured .md files after every task. Each run builds on the last. The system gets faster and smarter with every iteration — but only if knowledge is captured deliberately.
These principles are not guidelines — they are constraints.
Violating one breaks the guarantees the system depends on.
The workflow enforces them structurally, not through willpower.
