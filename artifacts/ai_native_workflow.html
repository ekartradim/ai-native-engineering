Workflow

6 stages
cursor · subagent-driven
continuously improving
01
Brainstorming
Activates before writing any code
ideate
Process
Refines rough ideas through targeted questions
Explores alternatives and trade-offs
Presents design in sections for validation
Output
design-document.md
Handoff condition
Design approved by human
approved design →
02
Plan
Activates with approved design document
decompose
Process
Decomposes work into small, executable tasks
Each task specifies exact file paths
Complete code and verification steps per task
Output
plan.md
Handoff condition
Plan reviewed and confirmed
confirmed plan →
03
Work
Activates with plan · two execution modes
execute
Mode A — subagent-driven
Fresh subagent dispatched per task
Two-stage review: spec compliance → code quality
Isolated context per task, no bleed-over
Mode B — batched execution
Tasks executed in batches
Human checkpoint between each batch
Review before proceeding to next batch
during implementation →
04
Test-driven development
Activates during implementation · enforced loop
verify
Red → Green → Refactor
Write failing test · watch it fail (RED)
Write minimal code · watch it pass (GREEN)
Commit · then refactor (REFACTOR)
Enforcement
Code written before tests is deleted
No skipping the RED phase
No committing on failing tests
between tasks →
05
Code review
Activates between tasks
gate
Process
Reviews implementation against plan spec
Issues reported and categorised by severity
Gate rule
Critical issues block all forward progress
Must resolve before next task begins
after task · optional →
06
Compound optional
Captures lessons learned · accelerates future runs
learn
Process
Extracts reusable lessons from completed task
Saves to structured .md folder system
Organised for future retrieval and reuse
Goal
Each run faster and better than the last
Institutional knowledge compounds over time
lessons/*.md
Steps 04 and 05 run in a continuous loop during step 03.
Step 06 is optional but recommended after every completed task.
