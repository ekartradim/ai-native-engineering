Compound Engineering Systems

Plan + Compound Memory System

Two-phase architecture

1

Capture

/ce-compound

Writes one structured file per solved problem. Mandatory YAML frontmatter with enum-validated fields. The frontmatter is the search interface.

↓
2

Retrieval

/ce-plan → learnings-researcher

Auto-spawns on every planning session. Greps frontmatter → scores candidates → full-reads only strong matches (3–7 files).

Token economics

Approach → tokens used

Load all solutions into context ~50,000+
Grep → frontmatter → score → read 500–2,000
25–100× less context consumed

Key insight

The YAML frontmatter schema is the retrieval mechanism — a grep-based search index designed for ripgrep. No vector DB, no embeddings, no external service.
