Compound Engineering Key Principles
Agent-native mindset
The agent is your equal
1
Environmental parity
If you can run tests, check logs, or open a PR, so should the agent. Anything withheld becomes manual work for you.
2
CLAUDE.md is the memory
Context compounds over time. Every solved problem becomes searchable knowledge for the next session.
3
Extract your taste into the system
What lives in senior engineers' heads should live in files the agent reads, not in code review.
Beliefs to let go
Your job is to ship value
1
Not every line needs review
If you do not trust the output, fix the system. Do not compensate by doing everything manually.
2
Code is not the primary artifact
A system that consistently produces good code beats any single brilliant implementation.
3
The 50/50 rule
Half your time building features, half improving the system. One hour on a review agent saves ten hours of review.

Auto-spawns on every planning session. Greps frontmatter → scores candidates → full-reads only strong matches (3–7 files).

Token economics

Approach → tokens used

The YAML frontmatter schema is the retrieval mechanism — a grep-based search index designed for ripgrep. No vector DB, no embeddings, no external service.

AI Native Engineering · Meta Skill
How to Create a Skill
Teach Claude once. It replicates forever.
1
Define intent
What should this skill enable Claude to do? When should it trigger? Clear trigger description prevents undertriggering.
2
Write SKILL.md
Instructions the agent reads on trigger. Under 500 lines. Add scripts/ for reusable code, assets/ for templates.
3
Write test cases
2 to 3 realistic, messy prompts. Write how a real user would ask, not a sanitised ideal.
4
Run evals in parallel
One agent runs with the skill, one runs without. Grade the diff. Repeat per test case simultaneously.
5
Iterate until satisfied
Read feedback, improve SKILL.md, rerun. Generalise fixes across all prompts, not just the failing one.
↻ loops back to step 4
6
Package and ship
Run package_skill.py to validate and zip. Produces a .skill file ready to install.
A skill is a replicator block. Teach it once, it assembles itself into every future task.
