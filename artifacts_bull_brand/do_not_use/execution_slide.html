What it actually solves

Problem 1

Context degradation

Long coding sessions lose coherence. ce-work dispatches subagents with fresh context per unit — task 10 gets the same quality as task 1.

Problem 2

Hidden dependencies

System-Wide Test Check pauses before each task:

  • What callbacks/middleware fire when this runs?
  • Do tests exercise the real chain, not just mocks?
  • Can failure leave orphaned state?
  • What other interfaces expose this?

Problem 3

Scope creep

Scope Boundaries (explicit non-goals) and Requirements Trace keep execution anchored to the original request. Deferred questions are tracked and resolved, not forgotten.

Problem 4

"80% done" syndrome

Task list with verification criteria per unit. Every task has a definition of done from the plan. Progress is visible, not claimed.

Key insight

ce-work's power isn't better code generation — it's enforced structure. The agent can't skip steps the way a human under pressure would. Every unit has verification criteria, every PR has a monitoring plan.
