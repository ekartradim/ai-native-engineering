CLAUDE.md is a suggestion.  Your linter is a law.
When AI ships more code than any human can review,
instructions aren't enough — you need deterministic guardrails.
The 4 Layers of Safety
Linters
Automatically flag when code breaks team conventions
automated code spell-checkers
ESLint  ·  Ruff  ·  RuboCop  ·  Clippy  ·  golangci-lint
Complexity Limits
Prevent code from becoming too tangled to understand or change
rules that force code to stay simple
complexity  ·  max-depth  ·  max-lines-per-function  ·  cognitive-complexity
CI / Static Analysis
Catch bugs, security holes and visual regressions before they ship
automated quality gates before code ships
Playwright  ·  SonarQube  ·  CodeQL  ·  Semgrep  ·  DryRun Security
Property-Based Testing
Define what must always be true, then hammer it with random inputs
robots trying to break your code
Hypothesis  ·  fast-check  ·  QuickCheck  ·  proptest
