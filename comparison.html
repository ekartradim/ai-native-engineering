Skills

Skills package reusable workflows and domain knowledge. They load into the main agent's context — no separate window.

CursorClaude Code
File formatSKILL.md with YAML frontmatterSame
Locations.cursor/skills/, ~/.cursor/skills/, also loads from .claude/skills/ as fallback.claude/skills/, ~/.claude/skills/
Invoked by/skill-name or auto-applied by agent/skill-name or auto-applied by agent
Disable auto-applydisable-model-invocation: true in frontmatterNot a frontmatter option — always context-eligible
Can include scriptsYes (scripts/, references/, assets/ subdirs)Yes (same structure)
Can spawn subagentsNo — skill is context only. Main agent decides to delegateWorkflow text uses Task(...) / Agent(...) — model emits the tool call
Install from GitHubSettings → Rules → Remote RuleVia plugin system or manual clone
Key difference: In Claude Code, plugin skills reliably emit Agent tool calls. In Cursor, a skill can only instruct the main agent in natural language; the model then decides whether to call the Task tool.
